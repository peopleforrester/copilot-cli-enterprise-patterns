**Time:** 60 minutes

## What you'll learn

- How to enter and exit Plan Mode (`Shift+Tab`)
- How to read a plan critically
- When to push back vs when to approve
- The difference between an in-session plan and a written spec
- When to skip planning

## Read first

- `patterns/plan-mode-workflow.md`

## The mental shift

Most engineers using a chat AI prompt → code → fix. Plan Mode forces prompt → plan → critique → code. The middle two steps are where the cost savings live.

A plan you can disagree with is cheaper than code you have to rewrite.

## Exit criteria

- [ ] You enter Plan Mode by reflex on multi-file changes
- [ ] You can read a plan and identify at least one weakness in under 60 seconds
- [ ] You know when *not* to plan
- [ ] You can explain the difference between a plan and a spec
