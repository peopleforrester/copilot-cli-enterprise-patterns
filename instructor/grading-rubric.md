# Grading Rubric

For instructor-led cohorts that need to certify completion. Self-paced learners can use it as a self-check.

## Foundations (Module 01) — 10 pts

| Criterion | Pts |
|---|---|
| Can recite the Four Pillars from memory | 3 |
| Can locate `AGENTS.md` and user `instructions.md` and explain precedence | 3 |
| Can explain Plan Mode in one sentence | 2 |
| Can name the default model | 2 |

## Context Management (Module 02) — 15 pts

| Criterion | Pts |
|---|---|
| Lab 01 completed: session recovered, no WIP lost | 8 |
| Can name three signals to `/clear` | 3 |
| Uses subagents for searches in observed practice | 4 |

## Plan Mode (Module 03) — 15 pts

| Criterion | Pts |
|---|---|
| Lab 02: final plan addresses all 10 hidden questions | 10 |
| Can articulate plan vs spec distinction | 3 |
| Knows when to skip planning | 2 |

## Externalization (Module 04) — 20 pts

| Criterion | Pts |
|---|---|
| Lab 03: AGENTS.md slimmed to ≤500 lines, nothing important lost | 10 |
| At least one item moved to a hook | 3 |
| Wrote a Skill that triggered correctly on first attempt | 5 |
| MIGRATION-NOTES.md is clear and complete | 2 |

## Verification (Module 05) — 20 pts

| Criterion | Pts |
|---|---|
| Lab 04: Pillar 4 wired from scratch, agent fixes the bug autonomously | 10 |
| Lab 05: completed both parts, can articulate the difference | 5 |
| Wrote a custom hook (security gate or test enforcement) outside the lab | 5 |

## Enterprise Hardening (Module 06) — 10 pts

| Criterion | Pts |
|---|---|
| Security review packet drafted | 5 |
| Onboarding script works on a fresh account | 3 |
| Can answer "Copilot CLI vs Claude Code" honestly | 2 |

## Capstone — 10 pts

| Criterion | Pts |
|---|---|
| Spec exists, matches final implementation | 3 |
| Each phase has its own commit | 2 |
| All tests pass with pristine output | 2 |
| Hooks ran on every edit (verifiable in transcript) | 2 |
| Code Review subagent used; findings addressed or deferred with rationale | 1 |

## Total — 100 pts

- **90+** — strong practitioner, ready to lead adoption on a team
- **75–89** — competent, can run unsupervised
- **60–74** — needs more reps; assign extra labs
- **<60** — re-run the modules they're weak in

Grading is calibration, not gatekeeping. The point is to know what each engineer needs more of.
